# Лабораторная работа 2

## Цель 

Научиться разрабатывать воспроизводимые аналитические инструменты. Студенту необходимо пройти полный цикл: от написания Python-скрипта для обработки бизнес-данных до его упаковки в Docker-образ и запуска в изолированной среде.

## Индивидуальное задание

Вариант № 12

Тематика данных - HR & Retention (Сотрудник, возраст, стаж, зарплата, уровень удовлетворенности, уволился (да/нет))

Техническое задание - Python + SQLite. Скрипт создает БД SQLite, таблицу по теме, вставляет 50 строк синтетических данных и выполняет SQL-запрос с агрегацией (GROUP BY).

## Описание компонентов

### Генератор данных

Файл с генератором - [generator.py]()

Скрипт запускается один раз на хосте перед docker compose up. Он создаёт файл hr_data.csv с 5 000 синтетическими данными о сотрудниках.

Поля:

| Поле | Тип | Описание |
|------|-----|----------|
| **id** | Целое число | Уникальный идентификатор сотрудника |
| **name** | Строка | Полное имя (фамилия + имя) |
| **age** | Целое число | Возраст сотрудника |
| **gender** | Строка | Пол |
| **position** | Строка | Должность (6 уровней: от младшего специалиста до директора департамента) |
| **experience** | Целое число | Стаж работы (в годах) |
| **salary** | Целое число | Зарплата (в рублях) |
| **performance** | Дробное число | Производительность работника (1–5) |
| **satisfaction** | Дробное число | Удовлетворённость работой (1–5) |
| **left** | Строка | Уволился или нет |

В файле прописана следующая логика:
- должность должна зависеть от стажа
- зарпалата - от должности и стажа
- вероятность увольнения - от удовлетворенности и стажа

### Dockerfile

Докерфайл - [Dockerfile]()

Что делает файл:
1. Использует Python 3.10-slim 
2. Устанавливает curl, sqlite3 и sqlitebrowser для работы с базой данных
3. Создаёт непривилегированного пользователя для запуска приложения
4. Устанавливает Python-библиотеки из requirements.txt
5. Копирует два скрипта — loader.py (загрузка данных) и dashboard.py (дашборд)
6. Запускает Streamlit-дашборд на порту 8501

### ETL-загрузчик

Фвйл для загрузки данных - [loader.py]()

Логика работы:
1. Ждёт появления CSV-файла с данными о сотрудниках (до 30 попыток с интервалом 2 секунды)
2. Создаёт SQLite базу данных с таблицей employees 
3. Загружает в эту таблицу данные из CSV
4. Выполняет запрос с GROUP BY: группирует данные по статусу (работает/уволился), для каждой группы выводит количество, среднюю зарплату и среднюю удовлетворённость. Результат выводит в логи.

### Dashboard

Файл, формирующий дашборд в Streamlit - [dashboard.py]()

1. Подключается к SQLite базе с данными о сотрудниках
2. Загружает данные
3. Строит дашборд с:
   - Ключевыми метриками: общее количество сотрудников, сколько работают/уволились, среднюю производительность и удовлетворённость
   - Графиками:
     - Распределение уволившихся по должностям (горизонтальная столбчатая диаграмма)
     - Распределение уволившихся по уровню удовлетворённости (круговая диаграмма)
     - Процент увольнений по группам стажа (столбчатая диаграмма)

### Docker Compose

Файл - [docker-compose.yml]()

Он объединяет 3 сервиса:

- **loader** — загрузчик данных:
  - Запускает скрипт loader.py один раз 
  - Ждёт, пока появятся данные, и загружает их в SQLite

- **dashboard** — дашборд:
  - Запускает Streamlit приложение на порту 8501 (только после успешного завершения loader)
  - Отображает аналитику по данным из базы

- **sqlite-web** — веб-интерфейс для SQLite:
  - Работает на порту 8085
  - Позволяет смотреть и редактировать базу данных через браузер

 ### Файл .env

 ```
# Пути к файлам
DB_PATH=/app/data/hr.db
CSV_PATH=/app/data/hr_data.csv
```

Хранит настройки путей к файлам внутри контейнера, что позволяет легко менять пути без изменения скриптов. Используется всеми сервисами (loader, dashboard, sqlite-web) через переменные окружения.

### Аналитика

#### График № 1 - Распределение уволившихся по должностям

### Анализ увольнений по должностям

На графике видно зависимость между уровнем должности и количеством увольнений:

**Чаще всего увольняются:**
- **Младшие специалисты** и **Специалисты** — на эти позиции приходится больше половину увольнений. На эти позиции приходят неопытные сотрудники с целью получения опыта работы. После освоения новых навыков, они увольняются с целью поиска лучших условий. 

**Реже всего увольняются:**
- **Директора департаментов** и **Руководители групп**. С высоких должностей увольняются реже, так как сотрудники боятся потерять статус, достойную заработную плату. К тому же, обычно люди, занимающие высокие должности, уже давно работают в данной компании и более лояльны к ней. Такие сотрудники в основном уходят только если находят новый вариант с гораздо лучшими условиями.
  
**Закономерность:** Чем выше должность, тем ниже вероятность увольнения. Это подтверждает гипотезу, что опытные сотрудники на руководящих позициях более лояльны компании.

<img width="1170" height="557" alt="image" src="https://github.com/user-attachments/assets/37709031-1bb1-4563-a7df-8d8a4c5a3722" />

### Распрделение уволившихся по уровню удовлетворенности

- **Средний уровень (3-4)** — самая многочисленная группа уволившихся. Это сотрудники, которые вроде бы не испытывают сильного дискомфорта, но и не горят работой. Именно они составляют основную массу текучести — их относительно легко терять, но сложно удержать без явных улучшений.
- **Низкий уровень (1-2)** — вторая по величине группа. Закономерно, что неудовлетворённые сотрудники уходят чаще — это ожидаемо и даже полезно для компании.
- **Высокий уровень (5)** — самая малочисленная группа. Довольные сотрудники увольняются редко, возможно, из-за личных обстоятельств.
  
<img width="1198" height="604" alt="image" src="https://github.com/user-attachments/assets/543e8d63-9c4b-4e29-97eb-2b9562506e2b" />

### Увольнения по группам стажа

На графике видно четкую закономерность: чем выше стаж, тем ниже процент увольнений.

- **Пик увольнений (0-1 год)** — такие сотрудники увольняются чаще всего. Это их первый опыт работы по специальности, поэтому их ожидания от работы могут не совпасть с реальностью. К тому же такие сотрудники могут не пройти испытательный срок.
- **Минимум (20+ лет)** — долго работающие сотрудники реже меняют место работы. Они уже привыкли к данной компании, скорее всего значительно продвинулись по карьерной лестнице, работа их в целом устраивает. Приоритет для этой группы - стабильность.
